## 🔍 搜索工具优先级

**更新日期**：2026-02-11

### 优先级顺序

1. **open-websearch**（首选）
   - ✅ 无需 API Key
   - ✅ 支持多引擎：Bing, Baidu, DuckDuckGo, Brave, Exa, GitHub, Juejin, CSDN
   - ✅ 免费使用
   - ✅ 中文搜索友好

2. **Tavily**（备选）
   - ⚠️ 需要 API Key
   - ✅ 搜索深度较好
   - ✅ 支持内容提取

3. **WebSearch**（内置）
   - ⚠️ 仅美国可用
   - 💡 紧急备选

### 使用建议

**搜索中文内容** → 必须用 open-websearch
**搜索技术文档** → open-websearch + Context7
**搜索最新资讯** → open-websearch（百度/必应）
**深度研究** → Tavily（如果 API Key 可用）

**更新日期**：2026-02-11

### 产品定位

**不是游戏，不是工具，而是互动内容平台**

> "让一切变得好玩"（Make Everything Playable）

核心理念：**内容不只是"被观看"，而是可以被"玩"**

### 技术架构

```
AI Coding + 多模态生成 + 硬件深度融合
    ↓
统一的互动 Runtime
    ↓
可调用手机所有传感器和输入设备
```

**硬件能力**：
- 麦克风（吹气检测、声音识别）
- 陀螺仪（摇晃检测、重力感应）
- 前后摄像头
- 触控屏（多点触控）
- 振动马达
- 其他传感器

### 创作模式

1. **手搓**：从零开始用自然语言创作
2. **魔改**：在已有创意基础上二次创作（Remix）

### 提示词最佳实践

**推荐结构**：
```
创建一个 [互动类型] 的互动内容
主题：[xxx]
玩法描述：[具体怎么玩]
互动反馈：[用户操作后的反馈]
AI点：[多模态/硬件调用]
```

**关键技巧**：
- ✅ 明确互动方式：**点一点、拖一拖、吹一吹、晃一晃**
- ✅ 描述具体反馈，不是抽象概念
- ❌ 不要写得太宽泛

### 真实用户反馈

#### ✅ 优点

1. **创作门槛极低**
   > "一个热梗、一张截图、一种情绪，甚至一个奇怪的念头，都可以成为创作源头"

2. **互动性强**
   > "和刷抖音一样，根本停不下来，太魔性了"

3. **硬件融合好**
   > "吹气、摇晃、触摸，各种玩法都有"

4. **Remix 文化**
   > "一键魔改，看到好玩的立即变成自己的版本"

#### ⚠️ 缺点

1. **生成速度慢**：平均 5 分钟/个
2. **稳定性待提升**：偶尔生成失败
3. **需要邀请码**：创作功能需要申请
4. **真实性不足**：有时逻辑不够严谨
5. **并发限制**：只支持一个任务

### 技术洞察

**核心难点**：Code（代码）与多模态生成的深度结合
- 代码负责交互逻辑和稳定性
- 多模态负责视觉张力
- 两者相互约束、相互影响

**Runtime 关键**：
- 统一接管用户输入（点击、拖动、传感器）
- 转化为结构化指令
- 在代码定义的边界内实时更新"世界状态"

### 创作案例

1. **"猫卜艺"** - 滑动吸猫 + 舒缓音乐
2. **"关你屁事"** - 过年应对亲戚追问的恶搞应用
3. **"风之谷"** - 吹气生成奇幻花草
4. **唱歌评分** - 根据唱功生成不同画面
5. **颠球游戏** - 陀螺仪控制乒乓球

### 行业意义

**代表新趋势**：
- 从"看内容"到"玩内容"
- AI Native 产品的典范
- 技术平权：让每个人都能把抽象念头变成具体互动

**与传统产品的区别**：
| 维度 | 短视频（抖音） | 游戏 | Loopit |
|------|--------------|------|--------|
| 交互 | 被动观看 | 复杂操作 | 简单互动 |
| 创作门槛 | 高 | 极高 | 极低 |
| 分享性 | 强 | 弱 | 强（可Remix）|
| 硬件融合 | 弱 | 中 | 强 |
