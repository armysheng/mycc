# MyCC 新成员入场清单

> 用途：新同学加入当天，按本清单完成环境与协作规范对齐。

## 1. 账号与权限

1. GitHub 仓库读写权限已开通。
2. 本机已安装 `git`、`node`、`npm`。
3. 已配置 SSH key 并可拉取仓库。

## 2. 本地目录与 worktree

1. 从 `main` 创建独立 worktree。
2. worktree 命名建议：`mycc_<name>`。
3. 分支命名遵循前缀规范：`codex/*` 或 `cc/*`。

示例：

```bash
git fetch origin --prune
git worktree add ../mycc_andy -b codex/andy-<topic> origin/main
```

## 3. 环境初始化（必须）

1. 复制后端 `.env`（从团队约定模板或现有可用环境）。
2. 安装依赖：前后端都执行 `npm install`。
3. 执行 build：前后端至少各跑一次 `npm run build`。
4. 确认端口不冲突并固定使用约定端口。

## 4. 开发前确认

1. 任务已在看板登记并指定 owner。
2. 验收标准已写清（至少包含 3 条可验证断言）。
3. 影响范围已确认（后端/前端/脚本/文档）。

## 5. 提交与 PR 规范

1. 小步提交，不混入无关文件。
2. PR 说明必须包含：范围、风险、回归、回滚。
3. 涉及接口变更，必须附接口示例或契约说明。

## 6. 每日交接（强制）

1. 分支 + 最新 commit SHA。
2. 今日完成项。
3. 未完成项与阻塞。
4. 明日第一步。

## 7. 禁止事项

1. 禁止直接在 `main` 目录做功能开发。
2. 禁止未沟通直接修改他人 owner 文件。
3. 禁止未过门禁直接合并。
